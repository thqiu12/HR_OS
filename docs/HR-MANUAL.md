# HR担当者 操作マニュアル

対象: 人事担当者 (法人HR/学校HR/校長/部門長)

## 1. 初回ログイン

1. 管理者から付与された ログインID + 初期パスワード でログイン
2. 右上のユーザーメニュー → **設定 / 2FA** から二要素認証を有効化（管理者は必須）
3. パスワード変更（自分のパスワードは `設定 / ユーザー` 自分の行 → リセット）

## 2. 採用フロー

### 2.1 求人を公開する
`採用管理 / 求人一覧` → 「求人を作成」
- 必須: 学校 / 部門 / 種別 (中途/新卒) / 公開状態
- 公開中にした瞬間 候補者が応募できる状態になる

### 2.2 候補者を取り込む
- 単件: `採用管理 / 候補者一覧` でチェックリストから手動追加
- 一括: `採用管理 / CSV インポート` で雛形に従って追加 (必須: name/kana/email/jobId)

### 2.3 選考を進める
候補者詳細で
- **次のステージへ** ボタン → 応募 → 書類選考 → 一次面接 → ... → 内定 → 入社済
- **面接設定** → 日時/場所/評価者を入力 → 候補者にメール通知 (テンプレート)
- **メール送信** パネルから 応募受付 / 内定 / 不採用 / 面接案内 を送信

### 2.4 内定通知
内定通知ボタン → 給与/開始日/期限を入力 → メール送信。同時に `onboarding_cases` が自動作成される。

## 3. 入社手続きフロー

### 3.1 内定者ポータル
`入社手続き / 案件一覧` → 詳細 → **コピー** ボタンで内定者ポータルURLをコピー、内定者にメールで送信。
内定者はログイン不要でURLから書類提出できる (招待トークンは7日有効)。

### 3.2 書類確認
案件詳細
- 各書類が「提出済」になったら → 「HR確認開始」 → 「承認」 or 「差戻し (理由必須)」
- 全書類「完了」になると 案件ステータスが自動で「完了」に

## 4. 評価フロー (8段階)

`評価管理 / 評価カルテ` → 社員選択 → **構造化評価を開始**

```
1. 目標設定 ── 上司+本人で MBO 目標を合意 (合計100%)
2. 中間面談 ── 上司が進捗を記入
3. 自己評価 ── 本人が各項目に5段階+所見
4. 一次評価 ── 上司が各項目に5段階+所見
5. 二次評価 ── 部門長/校長が確認・調整
6. 評価会議 ── HR が部門全体の分布を見て確定ランク調整
7. フィードバック面談 ── 上司↔本人 (実施日記入)
8. 確定 ── 給与改定/賞与係数に反映
```

### 4.1 評価項目
| カテゴリ | ウェイト | 内容 |
|---|---|---|
| 業績 (MBO) | 60% | 期初に設定した目標 (任意数, ウェイト合計100%) |
| 能力 | 30% | 専門スキル / リーダーシップ / コミュニケーション / 問題解決 (4固定) |
| 行動・価値観 | 10% | 協調性 / 主体性 / 倫理観 (3固定) |

### 4.2 評価会議 (キャリブレーション)
`評価管理 / 評価会議` で部門単位の分布を確認:
- 目安: S=10% A+=15% A=20% B=35% C=15% D=5%
- 偏りがある場合は個別ランクを調整

## 5. 組織管理

| ページ | 権限 | できること |
|---|---|---|
| 組織ツリー | 全員閲覧 | 法人→学校→部門の階層表示 |
| 社員一覧 | scope内 | 検索/フィルタ/CSV取込 |
| 部門マスタ | HR | 部門の追加/編集/削除 |
| 学校マスタ | group_admin/entity_hr | 学校の追加/編集/削除 |

## 6. リマインダー

`リマインダー` で 以下を表示・対応:
- 在留カード期限切れ (30日以内)
- 試用期間終了 (2週間〜1ヶ月以内)
- 入社書類差戻し中
- 雇用契約終了

「対応する」ボタンで該当タスクを既読化。

## 7. よくある質問

**Q. パスワードを忘れた**
A. group_admin に依頼 → `設定 / ユーザー` → 該当ユーザー → パスワードリセット

**Q. 退職社員のデータは消えない?**
A. status=「退職」にしても5年間は監査ログ用に保持。マイナンバー等PIIは退職5年経過で自動消去 (`/api/cron/retention`)。即時消去が必要な場合は HR に依頼。

**Q. 監査ログは改ざんできる?**
A. SQLite トリガで UPDATE/DELETE をブロック + ハッシュチェーン。`設定 / 監査ログ / 整合性検証` で常時検証可。

**Q. 社員にメール通知が届かない**
A. `設定 / API使用量` → email logs で配信状態確認。bounced の場合はアドレス再確認。
